### Correction 1: Makefile — Reverted from `-fPIE` to `-fno-pic`

**File: `sprx-plugin/Makefile`**

| Before | After | Rationale |
|--------|-------|-----------|
| `-fPIE` | `-fno-pic` | PSL1GHT sprxlinker is fragile and fails to parse PIE binaries. It rigidly expects a statically-mapped **ET_EXEC** ELF to successfully generate the `.lib.stub` NID import tables. |
| `-Wl,-q` (`--emit-relocs`) | **Removed** | Cobra's VSH loader uses its own internal relocation mechanism — explicit emit-relocs is unnecessary and can confuse the linker pipeline. |

**Result:** CFLAGS restores `-fno-pic`. LDFLAGS has no `-Wl,-q` or `--emit-relocs`.

### Correction 2: Prevent CRT Override — Renamed Entry Points

**File: `sprx-plugin/main.c`**

The authoritative initialization object `/usr/local/ps3dev/ppu/powerpc64-ps3-elf/lib/lv2-sprx.o` (mandated by the SDK) already defines the global `_start` and `_stop` routines. Defining them again in main.c bypasses the C Runtime (CRT), leaving the Table of Contents (TOC) pointer in register `r2` filled with garbage data.

| Before | After |
|--------|-------|
| `int _start(u64 args)` | `int ldtoypad_start(u64 args)` |
| `int _stop(void)` | `int ldtoypad_stop(void)` |

**Result:** No symbol collision with lv2-sprx.o. The CRT's `_start` sets up `r2` (TOC), then calls our `ldtoypad_start`.

### Correction 3: Removed SYS_PROCESS_PARAM_FIXED Macro

**File: `sprx-plugin/main.c`**

**Deleted:** `SYS_PROCESS_PARAM_FIXED(1001, 0x4000)`

**Reason:** This macro inherently injects the `.sys_proc_param` section, which marks the binary as a standalone **GameOS executable (EBOOT)**. When Cobra detects this marker, it rejects the file as a container mismatch, preventing VSH injection.

### Correction 4: Mathematically Correct 64-bit ABI Assembly Block

**File: `sprx-plugin/main.c`**

Because we removed `SYS_PROCESS_PARAM_FIXED`, we must manually supply:
- `.sys_proc_prx_param` — module parameter information
- `.rodata.sceModuleInfo` — module metadata (critical for Cobra validation)
- `.lib.ent` — entry/exit hook table

**CRITICAL:** `.quad` (8 bytes) is used instead of `.long` (4 bytes) for function descriptor references, properly preserving the full 64-bit PowerPC function descriptor triplets `{code pointer, TOC pointer, environment pointer}`.

| Directive | Size | Correct on PPC64? |
|-----------|------|--------------------|
| `.long ldtoypad_start` | 32-bit | **NO** — truncates the 64-bit descriptor |
| `.quad ldtoypad_start` | 64-bit | **YES** — preserves full descriptor address |

**Assembly block structure:**
1. `.sys_proc_prx_param` — 40-byte parameter header with `__libentstart/end` and `__libstubstart/end` references
2. `.rodata.sceModuleInfo` — Cobra validation metadata ("ldtoypad" module name, version 1.1)
3. `.lib.ent` entry hook — `0x01300000` start identifier + `.quad ldtoypad_start`
4. `.lib.ent` exit hook — `0x01400000` stop identifier + `.quad ldtoypad_stop`

### Key Architectural Principles (Final)

1. **`-fno-pic`**: Statically-mapped ET_EXEC — sprxlinker can resolve NID tables for `.lib.stub`
2. **Hand-rolled asm with `.quad`**: Correct PPC64 ABI — no 32-bit truncation of function descriptors

5. **Extra LDD only**: No LV2 syscall patching. No `sys_lv1_peek`/`sys_lv1_poke`. No kernel panic risk.
6. **One-shot enable token**: Consumed on first boot — next reboot is safe. No infinite boot loops.
